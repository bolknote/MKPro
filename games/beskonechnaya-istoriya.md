# Бесконечная история

- Авторы: Владимир Талалаев, Владимир Шило
- Источник: журнал *Техника - молодёжи*, № 1, 1988, рубрика «Клуб электронных игр», статья «Бесконечная история», с. 59-61; выпуск описан на [FantLab](https://fantlab.ru/edition387762)
- Программа: [beskonechnaya-istoriya.txt](beskonechnaya-istoriya.txt)
- Листинг выверен по страницам 59-61 журнала из DjVu-скана с помощью `djvu2pdf`, macOS Vision OCR и ручной сверки таблицы.

## Сюжет и цель

Игра сделана по мотивам фильма «Бесконечная история». Пустота шаг за шагом поглощает страну Фантазию; если она доберётся до дворца Принцессы, страна погибнет. Игрок начинает в самом центре страны, во дворце Принцессы, и отправляется искать средство для её исцеления.

Цель - найти Южного Оракула. За каждый ход можно продвинуться только на одну позицию, а вокруг продолжают действовать Пустота, Гморг, Сфинкс и Фалькор.

Игроку известно следующее:

1. Южный Оракул находится где-то на юге.
2. Подступы к нему охраняет Сфинкс.
3. Напрямую через Сфинкса не прорваться.
4. Если идти на север, при удаче можно встретить Фалькора: он перенесёт игрока на широту Южного Оракула, минуя Сфинкса.
5. На севере поджидает Гморг, который способен сожрать игрока.
6. Тьма-Пустота иногда ставит ловушки, из которых нельзя выбраться.
7. Если не успеть к Оракулу, Пустота сомкнётся.
8. Если достигнуть Оракула, программа остановится: страна Фантазия спасена.

## Исходные данные

После ввода программы занести в регистры:

```text
31 хП7
99 хП8
10 хП9 хП1
1 хП3
100 хП0 хПА хПС
7 /-/ хП6
40 /-/ хПВ
39 /-/ хПЕ
Сх хП4 хП5 хП2 ÷ ВП хПД
```

Затем прочистить стек: нажать `0` и семь раз `В↑`. Запуск: `В/О С/П`.

## Управление

Движение задаётся переключателем углов `Р-ГРД-Г`.

| Положение | Действие за один ход |
| --- | --- |
| `Г` | поворот на 90 градусов по часовой стрелке и шаг в новом направлении |
| `Р` | поворот на 90 градусов против часовой стрелки и шаг в новом направлении |
| `ГРД` | шаг в прежнем направлении |

Текущие координаты выводятся в режиме мерцания. Сначала мерцает координата `X` - долгота: положительные числа означают восток, отрицательные - запад. Затем выводится координата `Y` - широта: минус означает юг, плюс - север.

## События

Если игрок попал в зону, захваченную Гморгом, в область, простреливаемую Сфинксом, или если Пустота сомкнулась и поглотила дворец Принцессы, на экран выводится Тьма.

Если игрок оказался на других участках, захваченных Пустотой, на индикаторе вспыхнет `ЕГГОГ`. В этом случае нужно выбрать новое положение переключателя углов и нажать `В/О С/П`. Если за пять ходов не удалось выбраться из «болота ЕГГОГов», игрок угодил в ловушку Пустоты и проиграл. Во время блуждания по этому болоту текущие координаты не выводятся.

При выходе к Южному Оракулу программа останавливается, а в регистрах `X` и `Y` находятся нули. В статье это трактуется как благополучное возвращение в замок Принцессы с победой.

Минимальное время до выигрышного финала - 15-20 минут. Максимальное время до полной победы Пустоты над страной - около двух часов.

## Регистры

| Регистр | Назначение |
| --- | --- |
| `R7`, `R8` | адреса переходов |
| `R9` | необходимая константа |
| `RA` | число для управления движением |
| `R4`, `R5` | текущие координаты `X` и `Y` |
| `R2`, `R3` | направление перемещения; в начале в одном из них должен быть нуль, в другом - единица |
| `R0`, `R1` | движение Тьмы-Пустоты |
| `RD` | сама Тьма |
| `R6` | долгота границы зоны, патрулируемой Фалькором |
| `RB` | широта Южного Оракула |
| `RE` | широта Сфинкса, на единицу больше широты Оракула |

## Блоки программы

| Адреса | Назначение |
| --- | --- |
| `00`-`30` | управление поворотами при движении |
| `31`-`41` | управление движением Пустоты |
| `42`-`49` | управление движением игрока по территории страны |
| `50`-`63` | проверка попадания в Пустоту и к Гморгу |
| `64`-`77` | проверка попадания в зону, патрулируемую Фалькором |
| `78`-`84` | проверка попадания в зону, простреливаемую Сфинксом |
| `85`-`95` | проверка попадания к Южному Оракулу |
| `96`-`A5` | индикация текущих координат и возврат на начало программы |

## Примечания к листингу

Листинг в журнале дан для МК-52 и занимает адреса `00`-`A5`. В записи программы здесь используются принятые в репозитории обозначения: журнальное `ИПn` записано как `Пхn`, а журнальное `Пn` в командах записи в регистр - как `хПn`.

Проверенные места, где скан читается неочевидно:

| Адрес | Команда |
| --- | --- |
| `18` | `Кx≠07` |
| `31` | `FL0` |
| `33` | `FL1` |
| `54`, `59`, `74` | `Fx>=0` |
| `69`, `81`, `88`, `93` | `Fx=0` |
| `57`, `65` | `FВx` |
| `63` | `К-` |
| `97` | `КПП8` |
| `99`-`A4` | `В↑` |
| `A5` | `В/О` |

## Варианты сокращения из журнала

На странице 61 журнала приведён отдельный сокращённый вариант для Б3-34. В первоначальном варианте для Б3-34 Сфинкс отсутствовал из-за нехватки памяти; после сокращения блока `00`-`30` освободилось шесть команд, и Сфинкс был добавлен. Исходные данные для этого варианта остаются прежними, кроме регистров `R7` и `R8`: `25 хП7`, `93 хП8`. Регистр `RE` не используется, потому что широта Сфинкса задаётся программно.

В той же заметке предлагаются дополнительные приёмы сокращения для этого сокращённого варианта Б3-34:

| Фрагмент | Правка | Эффект |
| --- | --- | --- |
| `03`-`20` | переставить фрагменты `12`-`15` и `16`-`20`, одновременно сменив адреса переходов | подготавливает дальнейшее освобождение регистра `R7` |
| `16`-`17` | заменить на `16 Fx≠0`, `17 25`; в команде `04` поставить новый адрес `16` | освобождается регистр `R7` |
| `32`-`33` | вместо умножения через константу `10` в `R9` записать `32 ВП`, `33 1` | освобождается регистр `R9` |
| `54`-`57`, `86`-`89` | записать `29` в освобождённый `R9`; первый блок заменить командой `Кx<09`, второй - `Кx≠09` | экономия шести команд |
| старые `64`-`65`, `69`-`70` | заменить прямые условные переходы косвенными с использованием `R7` | ещё две команды экономии, всего восемь |

Далее автор предлагает при желании убрать ещё одну-две команды: записывать в `RD` вместо «пустышки» настоящую Тьму или `ЭГГОГ`; во втором случае отпадает отдельная остановка `С/П`, а «сверхчисло» можно использовать и для косвенного перехода.

Эти правки не применены к текущему файлу [beskonechnaya-istoriya.txt](beskonechnaya-istoriya.txt): здесь сохранён основной журнальный листинг для МК-52.
