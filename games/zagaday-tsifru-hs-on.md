# Загадай цифру

- Автор и публикатор: `Hs On`
- Опубликовано: 14 июля 2026 года в Telegram-чате «МК61 / МК52 / МК85»
- Программа: [zagaday-tsifru-hs-on.txt](zagaday-tsifru-hs-on.txt)
- Исходник MKPro: [zagaday-tsifru.mkpro](../examples/zagaday-tsifru.mkpro)

## Описание

При каждом ходе игрок загадывает цифру от `0` до `7` и вводит её в ПМК. К этому моменту ПМК уже вычислил собственное предсказание. Если цифры совпали, счёт игрока уменьшается на 7 очков; если ПМК ошибся, счёт увеличивается на 1 очко.

Задача игрока - придумывать последовательность цифр, которую программа не сможет предсказывать достаточно часто, и не дать счёту уйти в минус. Простые повторяющиеся закономерности программа постепенно запоминает.

## Подготовка

После ввода программы установить регистры:

| Регистр | Значение |
|---|---|
| `Р5` | `0` |
| `Р7` | `ЕЕ`: `188 В↑ 166 К∨ K{x} ВП 2` |
| `Р8` | `8,NNNNNNN`, где каждая `N` - произвольная цифра от `0` до `7` |
| `РВ` | `1111111,1` |
| `РС` | `10` |
| `РD` | `7,7777777` |

Нажать `В/О`, `С/П`. На индикаторе появится начальный счёт `0`.

## Ход игры

1. Ввести загаданную цифру от `0` до `7`.
2. Нажать `ПП`. ПМК покажет своё заранее вычисленное предсказание со знаком минус: от `0` до `-7`.
3. Нажать `С/П`. На индикаторе появится обновлённый счёт, после чего начинается следующий ход.

Игра может продолжаться неограниченно долго или до заранее выбранной границы, например `+100` либо `-100`. Для новой игры достаточно записать `0` в `Р5` и нажать `В/О`, `С/П`.

## Как устроено предсказание

Программа действительно реализует компактную однослойную нейронную сеть: три независимых бинарных пороговых нейрона без скрытого слоя. Каждый нейрон предсказывает один разряд числа: `4`, `2` или `1`.

На вход каждого нейрона поступает история семи ходов. Каждая цифра `0..7` представляется тремя битами, поэтому нейрон получает 21 бинарный вход. Его 21 бинарный вес хранится как семизначная восьмеричная маска. Операция `К⊕` сопоставляет входы с весами, затем программа подсчитывает число единичных бит и сравнивает его с порогом `10`. Поскольку входов 21, это проверка большинства без возможности ничьей. Три результата образуют предсказанную цифру.

Это бинарные перцептроны в необычной записи: вместо вещественных коэффициентов используются два возможных направления веса, закодированные битами маски, а сумма вычисляется через расстояние Хэмминга.

Обучение выполняется после каждого хода, на котором соответствующий выходной бит предсказан неверно. Программа выбирает случайную часть весов и переворачивает необходимые биты операцией `К⊕`, приближая маску к входному образцу для правильного ответа. Это стохастическое обучение по ошибке, но не обратное распространение ошибки.

Формула обновления весов:

```text
Веса := Веса XOR ((Входы XOR Веса XOR Маска*загаданного)
                  AND Маска*несовпадения
                  AND Случайная_маска)
```

## Регистры

| Регистр | Назначение |
|---|---|
| `Р0` | счётчик цикла |
| `Р1-Р3` | веса нейронов для битов `4`, `2`, `1` |
| `Р4` | косвенная адресация |
| `Р5` | счёт игры |
| `Р6` | цифра игрока плюс `10`, в формате `1N` |
| `Р7` | адрес косвенного перехода `ЕЕ` |
| `Р8` | история ходов в формате `8,NNNNNNN`, от новых к старым |
| `Р9` | выделение битов цифры игрока |
| `РА` | обработка битов предсказания ПМК |
| `РВ` | константа `1111111,1` |
| `РС` | порог `10` |
| `РD` | константа `7,7777777` |
| `РE` | косвенная адресация |

## Структура программы

| Адреса | Назначение |
|---|---|
| `00-35` | вычисление предсказания: наложение масок весов, подсчёт битов, сравнение с порогом и сборка результата в `РА` |
| `36-41` | первая часть ввода-вывода |
| `42-47` | начало подпрограммы разложения числа на биты, вызываемой по адресу `ЕЕ` |
| `48-53` | продолжение подпрограммы для получения маски `0,0000000` либо `7,7777777` |
| `54-61` | продолжение ввода-вывода |
| `62-69` | обновление счёта; используется особенность `Kmax`, считающая ноль наибольшим числом |
| `70-95` | стохастическое обновление весов |
| `96-A4` | добавление нового хода в историю |
